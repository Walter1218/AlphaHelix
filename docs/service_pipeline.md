# AlphaHelix 服务链路规范

## 1. 两阶段选股架构

```
全市场 ~5300 只
    ↓ 股票池筛选（市值、流动性、非ST）
Universe ~2000 只
    ↓ 第一阶段：召回（规则/因子过滤）
召回池 ~50-200 只
    ↓ 第二阶段：排序（模型打分）
Top-N 输出
```

## 2. 两阶段定义

### 2.1 召回阶段

**目标**：从 Universe 中筛选出"可能好"的股票，缩小候选池

**方法**：
- 基于规则/因子的快速筛选
- 不使用 ML 模型
- 目标是高召回率（不漏掉好股票），而非高精确率

**当前实现**：
```python
def recall_multi_factor(df, n=200):
    df['recall_score'] = (
        df['mom_20'].rank(pct=True) * 0.3 +
        df['roe'].rank(pct=True) * 0.2 +
        df['net_mf_ratio'].rank(pct=True) * 0.2 +
        df['volatility_20'].rank(pct=True) * (-0.1) +  # 低波动更好
        df['dv_ratio'].rank(pct=True) * 0.2
    )
    return df.nlargest(n, 'recall_score')
```

**召回池大小**：
- 保守：50 只（胜率高，但可能漏掉好股票）
- 平衡：100 只
- 激进：200 只（召回率高，但排序压力大）

### 2.2 排序阶段

**目标**：在召回池内对股票进行精确排名

**方法**：
- 使用 ML 模型（Ridge/LR/LambdaRank/DNN）
- 训练数据：全量历史数据（大样本）
- 预测数据：仅召回池内的股票

**当前实现**：
```python
# 训练：用全量数据
model.fit(X_train_all, y_train_all)

# 预测：只对召回池内股票
recalled = recall_fn(test_df)
```

## 3. 两阶段指标体系

### 3.1 服务链路指标（最终输出）

| 指标 | 定义 | 计算方式 |
|---|---|---|
| **服务胜率** | Top-N 中超额收益>0 的比例 | `(t10['excess_return'] > 0).mean()` |
| **服务累计超额** | Top-N 组合的累计超额收益 | `t10.groupby('date')['excess_return'].mean().sum()` |
| **服务单调性** | 排名与胜率的 Spearman 相关 | 排名越高，胜率应该越高 |

### 3.2 召回阶段指标

| 指标 | 定义 | 计算方式 |
|---|---|---|
| **召回率** | 召回池中正收益股票占全市场正收益股票的比例 | `recall_pos / market_pos` |
| **召回精确率** | 召回池中正收益股票的比例 | `recall_pos / recall_total` |
| **召回提升** | 召回精确率 - 全市场正收益比例 | `recall_precision - market_precision` |

### 3.3 模型阶段指标

| 指标 | 定义 | 计算方式 |
|---|---|---|
| **模型胜率** | 模型排名 Top-1 在召回子集内的胜率 | 模型区分度 |
| **模型 IC** | 模型预测与实际收益的相关性 | Spearman 相关 |
| **模型单调性** | 排名越高，胜率应该越高 | 排名 1-10 的胜率趋势 |

## 4. 指标关系

```
服务胜率 = f(召回精确率, 模型胜率)
```

- **服务胜率** = 最终输出的胜率
- **召回精确率** = 召回池中好股票的比例
- **模型胜率** = 模型在召回池内的区分度

**关键洞察**：
- 模型胜率高不等于服务胜率高（召回池质量很重要）
- 召回精确率高不等于服务胜率高（模型区分度很重要）
- **两者都需要优化**

## 5. 当前最优配置

| 组件 | 配置 |
|---|---|
| 召回 | 多因子 Top-200 |
| 排序 | LR C=10.0, Top-25 特征 |
| **服务胜率** | **43.5%** |
| **模型胜率（Top-1）** | **67.5%** |

## 6. 改进方向

### 6.1 召回阶段改进

| 方向 | 预期效果 |
|---|---|
| 多因子综合召回 | 提升召回精确率 |
| 动态召回数量 | 根据市场状态调整召回池大小 |
| 行业均衡召回 | 避免行业过度集中 |

### 6.2 排序阶段改进

| 方向 | 预期效果 |
|---|---|
| 训练时模拟召回 | 训练数据只用召回子集 |
| 多模型集成 | LambdaRank + DNN Bagging |
| 特征工程 | 添加更多有效特征 |

### 6.3 联合优化

| 方向 | 预期效果 |
|---|---|
| 召回+排序联合训练 | 端到端优化 |
| 动态召回阈值 | 根据模型置信度调整召回池大小 |
