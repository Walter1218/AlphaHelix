# AlphaHelix 整合路线图：量化模型 + LLM 双引擎

> 文档日期：2026-07-07  
> 状态：Phase 1-4 已完成；数据穿越已修复；真实胜率 53-56%  
> 目标：把传统量化 ML 与 LLM 能力整合成一套可验证、可进化的 A 股选股系统

---

## 1. 执行摘要

AlphaHelix 已经跑通了数据层、因子层、评估层和 Agent 层，但核心预测能力尚未建立：

- 当前基于线性加权打分的 Pass2 无法产生稳定 alpha；
- 在线 Logistic 回归二分类在 5 日方向预测上失效；
- LLM 目前只用于最终报告生成，未充分发挥其在文本/事件/因子挖掘上的能力。

本方案提出**"量化模型为主、LLM 为辅"**的架构：

- **主引擎**：Gradient Boosting（LightGBM/XGBoost）做 cross-sectional 收益率/排序预测；
- **辅助引擎**：LLM 从新闻、公告、研报中提取情感/事件/叙事信号，并辅助因子代码生成；
- **决策层**：量化模型输出候选池与得分，LLM 做定性风控与叙事校验；
- **进化层**：用 IC/IR、分位数收益、回测超额驱动迭代。

## 2. 当前系统诊断

| 模块 | 现状 | 问题 | 改进方向 |
|---|---|---|---|
| 数据层 | Tushare 量价、估值、财务、资金、事件数据已接入 | 缺少北向、融资融券、龙虎榜、披露日预告、新闻 sentiment | 补齐另类数据 |
| 因子层 | 18+ 因子，rank 标准化后线性加权 | 无行业/市值中性化，原始资金金额量纲问题 | GBDT + 中性化 + LLM 文本因子 |
| 策略层 | 4 个硬编码策略 + regime 切换 | regime 映射僵化，几乎总是 event_driven | 用模型输出替代硬编码映射 |
| 预测层 | 在线 Logistic 回归二分类 | 5 日方向噪声大，模型未学到信号 | 改用 GBDT 回归/排序预测 |
| 持仓层 | Top-N 强制满仓 | 没有空仓机制，方向准确率失真 | 阈值制 + 空仓 + 滚动再平衡 |
| LLM 层 | 生成选股报告 | 未用于信号提取、因子生成、RAG | 接入情感/事件/RAG/多 Agent 风控 |
| 评估层 | 方向准确率、超额收益、最大回撤 | 缺少 IC/IR、分位数收益、交易成本 | 补全量化评估体系 |

## 5. 特征工程升级

### 5.1 数值因子处理

1. **截面 rank 标准化**：每只因子在当日截面上转换为 0~1 分位数。
2. **行业/市值中性化**：
   - 对每个因子，用当日截面回归去除行业和市值解释部分；
   - 保留残差作为纯净 alpha 信号。
3. **时序平稳化**：
   - 对动量类因子做分数差分或相对变化，降低非平稳性。
4. **移除问题因子**：
   - 去掉原始资金净流入金额（`net_mf_5d`、`net_mf_20d`）；
   - 保留 `net_mf_ratio`、`net_mf_divergence`。

### 5.2 LLM 因子

| 因子 | 输入 | LLM 输出 | 用途 |
|---|---|---|---|
| 情感得分 | 新闻、公告、研报 | -1 ~ 1 的情感分数 | 情绪面 alpha |
| 事件类型 | 业绩预告、政策、并购 | 标签（预增/暴雷/政策利好等） | 事件驱动 |
| 管理层语调 | 业绩说明会纪要 | 乐观/中性/谨慎 | 质量/预期因子 |
| 叙事风险 | 新闻、社交媒体 | 高波动/高杠杆/监管风险标签 | 风控过滤 |
| 行业景气 | 行业新闻、政策 | 景气度评分 | 行业配置 |

### 5.3 因子生成 Agent

参考 Microsoft RD-Agent：

1. LLM 阅读研报/论文/财报，提取潜在因子公式；
2. 自动生成 Python 代码并接入 `screen.py`；
3. 在 walk-forward 框架下计算 IC；
4. IC 显著的因子纳入主模型；不显著的淘汰。

## 6. 预测模型层

### 6.1 主模型：LightGBM / XGBoost

- **目标变量**：未来 5/10/20 个交易日相对沪深300 的超额收益（回归）。
- **备选目标**：未来收益率分位数（分类/排序）。
- **训练方式**：
  - 滚动训练：每月用过去 12~24 个月数据训练；
  - 或用 Qlib 式的 walk-forward 重训练。
- **特征输入**：数值因子 + LLM 因子。
- **样本权重**：
  - 按时间衰减，近期样本权重更高；
  - 或按流动性/市值加权。

### 6.2 为什么不用在线 Logistic 回归

- 5 日方向二分类噪声过大；
- 线性模型无法捕捉因子交互；
- GBDT 对异常值、缺失值、非线性更鲁棒；
- 行业验证 GBDT 是量化选股强基线。

### 6.3 在线学习（第二阶段）

在 GBDT 基线稳定后，再考虑：

- 增量更新 LightGBM（如 `lightgbm` 的 `refit`）；
- 或保留在线线性模型作为 GBDT 的残差修正。

## 7. 组合构建层

### 7.1 从预测到持仓

1. 每天对全市场候选股打分；
2. 按预测得分分 5/10 组，先验证 group 单调性；
3. 取 top-k 作为初始持仓池；
4. 应用约束：
   - 单一行业 ≤ 40%（市值权重）；
   - 单票 ≤ 20%；
   - 换手率惩罚，避免频繁调仓；
   - 预测得分低于阈值时空仓。

### 7.2 LLM 风控介入点

- **选股后**：对 top-k 候选做叙事风险、业绩暴雷、政策风险检查；
- **可执行动作**：
  - 否决某只股票；
  - 降低权重；
  - 增加风险提示。
- **不可执行动作**：
  - 凭空推荐未在候选池的股票。

### 7.3 交易成本

回测中必须扣除：

- 印花税 0.1% 单边（卖出）；
- 佣金 0.02% 双边；
- 滑点 0.05%~0.1% 双边。

## 8. 评估体系

### 8.1 预测质量

| 指标 | 说明 | 目标 |
|---|---|---|
| rank IC | 预测得分与未来收益的秩相关 | > 0.03 |
| ICIR | IC 均值 / IC 标准差 | > 0.3 |
| IC 衰减 | 多期 IC 持续性 | 衰减不要太快 |
| 分位数收益 | top 组 vs bottom 组收益差 | 单调且显著 |

### 8.2 组合绩效

| 指标 | 说明 | 目标 |
|---|---|---|
| 年化超额收益 | 相对沪深300 | > 5% |
| 信息比率 IR | 超额收益 / 跟踪误差 | > 0.8 |
| 最大回撤 | 组合净值高点到低点 | < 15% |
| 胜率 | 超额收益 > 0 的月份占比 | > 50% |
| 换手率 | 年均换手 | < 10 倍 |

### 8.3 过拟合控制

- Purged walk-forward CV；
- Deflated Sharpe Ratio；
- 训练集/验证集/测试集严格分离；
- 因子/模型调参只在训练集上做。

## 9. LLM Agent 角色重定义

### 9.1 alpha-analyst 职责

| 阶段 | LLM 任务 | 输出 |
|---|---|---|
| 数据层 | 不直接计算 | 调用工具获取数据 |
| 特征层 | 生成文本/事件因子 | 情感分数、事件标签 |
| 模型层 | 提出因子假设、生成代码 | 新因子代码 |
| 组合层 | 叙事风控、风险提醒 | 否决/降权建议 |
| 报告层 | 生成投资逻辑与报告 | Markdown/JSON |

### 9.2 多 Agent 风控（未来）

- **宏观分析师**：判断 regime、流动性、政策方向；
- **行业分析师**：评估行业景气度与竞争格局；
- **个股分析师**：解读财报、公告、新闻；
- **风控官**：检查集中度、止损、叙事风险；
- **量化引擎**：提供候选池与得分。

## 10. 实施路线图

### ✅ Phase 1：基线重建（2~3 周）

1. 改造 `screen.py`：
   - 输出截面 rank 因子；
   - 加入行业/市值中性化；
   - 移除原始资金金额因子。
2. 新增 `scripts/model_trainer.py`：
   - 用 LightGBM 训练未来 5/10/20 日超额收益回归模型；
   - 滚动训练 + walk-forward 预测。
3. 新增 `scripts/predictor_eval.py`：
   - 计算 rank IC、ICIR、分位数收益。
4. 改造 `walkforward.py`：
   - 用模型预测得分替代在线 Logistic 回归；
   - 支持阈值制空仓、行业约束、交易成本。

**Phase 1 实测结果（2026-07-05）**：

| 配置 | 结果 |
|---|---|
| 模型 | LightGBM 滚动训练（12 个月窗口，按月重训练） |
| 目标 | 未来 10 日相对沪深300 超额收益 |
| 特征 | 23 个 rank/行业市值中性化因子 |
| 样本 | 2024-01 ~ 2026-06，预测 106 个日期，8,222 条 |
| Rank IC（日均） | 0.0469 |
| ICIR | 0.238 |
| 正 IC 比例 | 59.4% |
| 分位数收益（0→4） | -0.20%, +0.21%, +0.34%, +0.64%, +0.38%（单调） |
| 组合（20 只等权，10 日再平衡，含交易成本） | 平均超额 +0.21%，累计超额 +28.13%，胜率 54.3%，换手率 63.7%/期 |

> 结论：Phase 1 验收通过。滚动 GBDT 显著优于静态因子加权（基准累计超额 -16.65%）。下一步进入 Phase 2 数据补齐。

### ✅ Phase 2：数据补齐（2 周）

1. 接入 `margin`、`moneyflow_hsgt`、`top_list`、`disclosure_date`；
2. 构建资金流/情绪类因子；
3. 验证新因子对 IC 的提升。

**验收标准**：新因子组合 rank IC > 0.03。

**Phase 2 实测结果（2026-07-05）**：

| 配置 | rank IC（日均） | 说明 |
|---|---|---|
| Phase 1 基线（23 个特征） | **0.0469** | LightGBM 滚动训练 |
| + margin / northbound（宏观） | -0.0012 | 宏观变量截面中性化后产生伪波动，反而稀释 alpha |
| + 仅 disclosure + top_list 个股特征 | 0.0235 | 略低于基线 |
| + disclosure 事件 dummy + 龙虎榜 5 日聚合 | **0.0345** | 最接近基线，但未超越 |
| XGBoost on v2 | 0.0207 | 未超越 |

关键发现：
- `days_to_disclosure` 单因子 IC 为 -0.062，`disclosure_near`（<=5 天）单因子 IC 为 +0.060，事件效应显著但极不稳定；
- 龙虎榜 5 日聚合因子整体为负 IC（-0.03 左右），可能意味着游资/机构异动后短期反转；
- 融资融券/北向资金为市场层面变量，直接加入截面模型会引入与市值/行业相关的伪信号，应作为宏观/交互变量另行处理。

> 结论：Phase 2 **未达预期提升**。基础量价/财务/预期因子已较完整，新增另类数据在这个窗口内未能稳定增强 alpha。建议：
> 1. 把宏观变量从截面特征中剥离，改为 regime/交互信号；
> 2. 把 `disclosure_near` 作为事件过滤/增强条件，而非普通特征；
> 3. 进入 Phase 3 引入 LLM 文本/事件因子，作为更独立的信号源。

### ✅ Phase 2.5：风控增强（1 周）

已尝试在 `portfolio_backtest.py` 中加入个股止损逻辑，并接入 `walkforward_gbdt.py`。

| 止损线 | 平均超额 | 累计超额 | 胜率 | 平均每期止损次数 |
|---|---|---|---|---|
| 无止损 | +0.28% | +39.50% | 52.4% | 0.00 |
| -3% | +0.10% | +11.80% | 49.5% | 6.69 |
| -5% | +0.17% | +22.16% | 51.4% | 3.90 |
| -7% | +0.22% | +30.30% | 49.5% | 2.32 |
| -10% | +0.24% | +32.00% | 52.4% | 1.11 |
| -15% | +0.26% | +36.82% | 52.4% | 0.27 |

**结论**：固定百分比止损在这个策略中**没有提升胜率**，反而因频繁触发止损、提前锁定亏损、现金闲置和交易成本而侵蚀收益。策略的 alpha 主要来自 **正偏分布**（少数大涨覆盖多数小跌），硬止损会破坏这一结构。

后续可尝试：
- 组合层面止损（如净值回撤超阈值减仓），而非个股硬止损；
- 动态止损（ATR、波动率自适应）；
- 事件/宏观过滤，避免在高波动 regime 满仓。

### ✅ Phase 3：LLM / 自动因子挖掘（2~3 周）

**Phase 3.1：LambdaRank + 复合因子（2026-07-05 已完成初步迭代）**

- 在 `model_trainer.py` 中加入 `lambdarank` 目标，直接优化“头部排在底部前面”。
- 加入 `scripts/factor_miner.py`，支持用 YAML 表达式批量评估因子 IC；
- 从候选表达式中筛出一批高 IC 复合因子：

| 因子 | 表达式 | 平均 IC | ICIR |
|---|---|---|---|
| defensive_quality | `roe / (volatility_20 + 1e-9)` | **+0.093** | +3.80 |
| smart_money_per_risk | `net_mf_ratio / (volatility_20 + 1e-9)` | +0.067 | +3.06 |
| quality_growth | `roe * profit_growth` | +0.067 | +3.67 |
| value_quality | `dv_ratio * roe` | +0.064 | +2.66 |

- 把复合因子集成进 `screen.py` / `build_dataset.py`，重新训练生产模型 `gbdt_latest_h10.lightgbm.txt`。

**回测结果对比（walk-forward, 20 只等权，10 日再平衡）**：

| 模型 | 平均超额 | 累计超额 | 胜率 | Top20 个股命中率 |
|---|---|---|---|---|
| 基线 GBDT（23 特征） | +0.21% | +28.1% | 54.3% | 48.2% |
| LambdaRank（23 特征） | +0.14% | +15.8% | 48.6% | 48.3% |
| **GBDT + 复合因子（30 特征）** | **+0.29%** | **+40.2%** | **53.3%** | **48.5%** |

> 结论：LambdaRank 单独没有提升 portfolio 收益，但复合因子显著增强了选股信号，累计超额从 28% 提升到 40%。胜率仍停留在 53% 左右，说明 10 日持有期本身方向胜率天花板较低，需要通过更长持有期或更强独立信号来突破。

**下一步（Phase 3.2）**：
1. 接入真实文本/事件数据源（新闻、公告、研报），用 LLM 提取情感和事件标签；
2. 把 `factor_miner.py` 与 LLM 连接：LLM 根据研报/论文生成表达式 → 本地 IC 评估 → 达标后自动加入因子库；
3. 尝试 20 日持有期模型，看胜率是否提高。

**20 日持有期测试结果（2026-07-05）**：

| 模型 | 平均 IC | 平均超额 | 累计超额 | 胜率 | Top20 命中率 |
|---|---|---|---|---|---|
| GBDT 回归 h20 | 0.0223 | -0.18% | -26.4% | 40.4% | 43.1% |
| LambdaRank h20 | 0.0411 | — | — | — | — |
| **GBDT 回归 h10（当前最优）** | **0.0527** | **+0.29%** | **+40.2%** | **53.3%** | **48.5%** |

> 结论：**20 日持有期并未提升胜率**，反而因信号衰减、波动放大导致 IC 和 portfolio 收益都下降。当前最优仍是 **h10 + 复合因子 + 回归**。胜率瓶颈不在持有期长度，而在独立信号源不足。

### Phase 3.2：更多另类数据 engineered 特征（2026-07-05）

新增脚本 `scripts/append_phase3_features.py`，在 Phase 2 v2 基础上加入：
- 融资融券余额及 5/20 日变化率（`margin_total_balance`, `margin_change_5d`, `margin_change_20d`）
- 北向资金当日/5日/20日净流入及 zscore（`northbound_net_today`, `northbound_net_5d_sum`, `northbound_net_20d_sum`, `northbound_net_20d_zscore`, `northbound_net_5d_vs_20d`）
- 龙虎榜 5日/10日窗口聚合（次数、累计净流入、平均 amount_rate / turnover_rate / pct_change）

宏观特征保留为日期维度原始值，不经过截面 rank/中性化；个股龙虎榜特征按 0 填充缺失。

**回测结果（walk-forward, h=10, 20 只等权）**：

| 数据集 | 特征数 | 平均 IC | 平均超额 | 累计超额 | 胜率 | Top20 命中率 |
|---|---|---|---|---|---|---|
| composite（仅复合因子） | 30 | **0.0623** | **+0.29%** | **+40.2%** | **53.3%** | **48.5%** |
| + Phase 2 v2（披露日 + 龙虎榜20日） | 40 | 0.0327 | — | — | — | — |
| + Phase 3（宏观 + 龙虎榜5/10日） | 58 | 0.0255 | -0.25% | -33.1% | 40.0% | 45.8% |

> 结论：**更多另类数据没有提升模型**。可能原因：
> - `disclosure` / `top_list` 信号本身不稳定，加入后反而稀释有效复合因子；
> - 宏观变量作为日期常数特征，使树模型过度按日期切片，降低泛化；
> - 龙虎榜 5日/10日 与已有 `smart_money_per_risk` 高度相关，信息冗余。
>
> 当前最优模型仍是 **composite-only（30 特征）**。另类数据更适合作为**事件过滤/宏观择时**使用，而非直接堆进截面模型。

#### 4.5 Walk-forward 阈值校准（已完成）

新增 `scripts/walkforward_threshold.py`：

- 完全基于历史数据校准阈值，避免在全样本上优化。

**回测结果（扣成本，等权 top-20，metric=win_rate）**：

| 6 期 | 0.30% | 41.2% | 54.3% |
| 12 期 | **0.36%** | **53.2%** | **55.2%** |
| 18 期 | 0.38% | 56.8% | 56.2% |
| 24 期 | 0.39% | 59.3% | 56.2% |

> 结论：walk-forward 阈值校准是**有效且未穿越**的改进。用过去 12 期（约 2.5 个月）数据选阈值，胜率从 53.3% 提升到 55.2%，累计超额从 40.2% 提升到 53.2%。训练窗口更长（18/24 期）效果略好，但提升边际递减，12 期作为默认参数已较稳健。

1. `walkforward_gbdt.py` 支持 `--use-wf-threshold --wf-metric win_rate`；
2. `retrain_gbdt.py` 支持 `--wf-threshold --wf-metric win_rate`，训练后会在模型同目录生成 `_threshold.json`；

#### 4.6 头寸加权 / 仓位优化（已完成）

新增 `portfolio_backtest.py` 的 `--weight-scheme` 参数，支持四种持仓权重方案：

| 方案 | 权重计算 | 说明 |
|---|---|---|
| `equal` | 等权 | 基线 |
| `score` | ∝ `max(0, predicted)` | 按预测置信度加权 |
| `risk_parity` | ∝ `1 / σ_20` | 按最近 20 日收益率波动率倒数加权 |
| `score_risk` | ∝ `predicted / σ_20` | 置信度与风险倒数结合 |

实现要点：
- 个股波动率完全基于缓存收盘价、决策日及之前的数据计算，无 look-ahead；
- 为加速风险平价计算，新增 `_close_by_code` 反向索引，把每次波动率计算从全缓存扫描降到 O(window)；
- `walkforward_gbdt.py` 同步支持 `--weight-scheme`。

**回测结果（扣成本，h=10，20 只，composite-only GBDT）**：

| 权重方案 | 平均超额 | 累计超额 | 胜率 | 年化夏普* |
|---|---|---|---|---|
| 等权 baseline | +0.29% | +40.15% | 53.3% | 0.50 |
| 按得分加权 | +0.37% | +44.77% | 49.5% | 0.42 |
| 风险平价 | +0.21% | +28.27% | 53.3% | 0.42 |
| 得分/波动率 | +0.23% | +25.05% | 48.6% | 0.30 |
| 等权 + WF 阈值 | +0.36% | +53.22% | **55.2%** | **0.61** |
| 得分加权 + WF 阈值 | +0.39% | +48.60% | 48.6% | 0.44 |

\* 年化夏普 = (平均超额 × 25.2) / (超额收益标准差 × √25.2)，按 10 日持有期年化。

> 结论：
> - **按预测得分加权**能提升平均/累计超额，但胜率下降、波动增大，风险调整后夏普更低；
> - **风险平价**直接压缩了高波动但高贡献股票的仓位，导致收益明显下降；
> - **得分/波动率**结合方案表现最差，说明直接用得分除以波动率会把噪声放大；
> - 与 walk-forward 阈值配合时，等权仍是风险收益比最优的选择。
>
> 因此 **生产链路默认仍使用等权 top-20**，`--weight-scheme score` 仅作为可选实验开关保留。后续若要做仓位优化，建议先尝试：
> 1. 行业/市值权重约束（把当前“行业数量上限”升级为“行业市值权重上限 + 风格中性”）；
> 2. 对 top 5 / top 10 集中仓位，而不是线性分散到 20 只；
> 3. 组合层面波动率缩放（宏观 regime 低迷时整体减仓），而非个股风险平价。

#### 4.7 行业/市值中性约束（已完成实验）

新增 `portfolio_backtest.py` 的两个可选风控开关：

- `--max-sector-weight <0.0~1.0>`：按行业总市值权重做集中度截断；
- `--neutralize-market-cap`：选股前对预测得分做截面回归，去除 `log(总市值)` 暴露后再排序。

总市值数据来自 Tushare `daily_basic.total_mv`，并已做缓存加速。

**回测结果（h=10，20 只等权，composite-only GBDT）**：

| 配置 | 平均超额 | 累计超额 | 胜率 | 年化夏普 | 换手率 |
|---|---|---|---|---|---|
| baseline（等权，无约束） | +0.29% | +40.15% | 53.3% | 0.50 | 65.7% |
| baseline + WF 阈值 | +0.36% | +53.22% | **55.2%** | 0.61 | 66.6% |
| + 行业权重上限 0.4 | +0.35% | +49.54% | 52.4% | 0.58 | 60.5% |
| + 行业权重上限 0.5 | +0.39% | **+56.86%** | 52.4% | **0.65** | 62.9% |
| + 行业权重上限 0.6 | +0.37% | +53.71% | 54.3% | 0.62 | 64.0% |
| + 行业权重上限 0.7 | +0.34% | +47.96% | 54.3% | 0.57 | 64.8% |
| + 市值中性化 | +0.05% | +1.05% | 49.5% | 0.09 | 65.4% |

> 结论：
> - **行业市值权重上限 0.5** 在 walk-forward 阈值基线上进一步提升了累计超额（+53.22% → +56.86%）和夏普（0.61 → 0.65），但**胜率没有提升**（52.4% vs 55.2%）。说明模型的一部分 alpha 来自行业/风格暴露，严格约束会牺牲方向胜率。
> - **市值中性化严重削弱收益**（累计超额从 +40% 掉到 +1%），说明当前模型的预测得分中市值/size 暴露本身就是有效信号，不能简单剔除。
> - 如果目标是**夏普/累计收益**，`--max-sector-weight 0.5` 可作为可选增强；如果目标是**胜率**，行业权重约束不是正确方向。
> - 后续若要做真正的行业中性，应先把行业因子从模型中**显式剥离**（训练时 target 对行业做回归残差），而不是在组合层简单截断。

新增 `scripts/macro_timing.py`，把 `apply_market_timing.py` 中的宏观仓位逻辑提取为可复用模块：

- 换仓成本按实际成交金额扣除。

**回测结果（h=10，20 只等权，composite-only GBDT + WF 阈值）**：

| 宏观择时 | 平均仓位 | 平均超额 | 累计超额 | 胜率 | 年化夏普 | 换手率 |
|---|---|---|---|---|---|---|
| 无（baseline） | 100% | +0.36% | +53.22% | **55.2%** | 0.61 | 66.6% |
| 连续缩放 | 92% | +0.33% | +48.62% | 54.3% | 0.54 | 66.6% |
| 二值 threshold=0.0 | 62% | +0.20% | +28.42% | 53.3% | 0.26 | 59.6% |
| 二值 threshold=-0.3 | 89% | +0.39% | **+60.03%** | 54.3% | **0.66** | 64.2% |
| 二值 threshold=-0.5 | 100% | +0.36% | +53.22% | 55.2% | 0.61 | 66.6% |
| + 行业权重上限 0.5 + threshold=-0.3 | 89% | +0.41% | **+62.48%** | 51.4% | **0.69** | 60.7% |

> 结论：
> - **连续缩放**因为现金拖累和信号偏弱，表现不如 baseline；
> - **二值过滤 threshold=-0.3** 在保持较高仓位（89%）的同时，累计超额从 +53.22% 提升到 +60.03%，夏普从 0.61 提升到 0.66，但**胜率没有提升**（54.3% vs 55.2%）；
> - 更激进的 threshold=0.0 仓位太低（62%），错过上涨，累计超额大幅下滑；
> - 与行业权重上限 0.5 叠加后，累计超额进一步提升到 +62.48%，夏普 0.69，但胜率降至 51.4%。
>
> 总体看，宏观择时能提升**风险调整后收益**，但尚未突破**胜率瓶颈**。这说明 A 股风格切换确实剧烈，简单的北向+融资融券阈值只能过滤部分极端负面环境，不能解决模型本身在震荡/切换期方向胜率低的问题。

#### 4.9 分 regime 训练多个 GBDT 模型（已完成实验）

新增 `scripts/walkforward_regime_gbdt.py` 与 `model_trainer.walk_forward_predict_by_regime`：
- 用 `macro_timing` 把每个交易日标注为 `strong / neutral / weak`；
- 滚动训练时，每个 fold 训练一个全局 fallback 模型 + 每个 regime 一个专用模型；
- 预测日根据当日 regime 调用对应模型；若该 regime 样本不足，回退到全局模型。

**回测结果（h=10，20 只等权，composite-only GBDT）**：

| 分 regime 配置 | WF 阈值 | 平均超额 | 累计超额 | 胜率 | 夏普 | 换手率 |
|---|---|---|---|---|---|---|
| 单一 GBDT baseline | 是 | +0.36% | **+53.22%** | **55.2%** | 0.61 | 66.6% |
| strong/neutral/weak (±0.3) | 是 | +0.03% | +1.01% | 49.5% | 0.05 | 66.0% |
| strong/neutral/weak (±0.1) | 是 | +0.23% | +31.86% | 51.4% | 0.42 | 69.2% |
| strong/neutral/weak (±0.1) | 否 | +0.17% | +21.58% | 51.4% | 0.30 | 67.8% |

> 结论：
> - **分 regime 建模当前显著劣于单一 GBDT**。原因可能是：
>   1. 各 regime 专用模型样本量大幅缩小（尤其 weak/neutral），容易过拟合；
>   2. 不同 regime 模型的预测得分尺度不一致，导致跨 regime 选股时 miscalibration；
>   3. 当前宏观信号（北向 20 日 zscore + 融资余额 5 日变化）对“风格/强弱”的划分不够稳定，regime 标签噪声大。
> - 后续若再做 regime 建模，应优先解决**跨模型分数校准**（如每个模型输出映射到统一分位数），并使用更稳定的 regime 定义（如市场波动率、信用利差、行业轮动速度等）。

#### 4.10 把宏观/风格状态作为交互特征（已完成实验）

新增 `scripts/append_regime_features.py`：
- 在 composite-only 特征集上追加 `regime_score`、三个 regime one-hot、三个宏观原始特征；
- 并生成显式交互项：`mom_20_x_regime`、`roe_x_regime`、`quality_growth_x_regime` 等。

用单一 GBDT 在新数据集 `features_h10_composite_regime.parquet` 上重新做 walk-forward 训练。

**回测结果（h=10，20 只等权，WF 阈值）**：

| 模型 | 特征数 | 平均超额 | 累计超额 | 胜率 | 换手率 |
|---|---|---|---|---|---|
| composite baseline | 30 | +0.36% | **+53.22%** | **55.2%** | 66.6% |
| composite + regime 交互特征 | 52 | -0.09% | **-17.48%** | 41.9% | 68.0% |

> 结论：把宏观/风格状态作为**截面特征或交互项**进入单一 GBDT，效果比分 regime 建模还差。原因：
> - `regime_score` 是日期常数，树模型会按日期切片，记忆特定时期的伪模式；
> - 交互项虽然随个股变化，但其系数由同一日期的宏观信号统一缩放，本质仍是按日期加权；
> - 这与 Phase 3 结论一致：**宏观变量不适合直接堆进截面模型**，应作为组合层面的择时/风控信号。

## 11. 风险与应对

| 风险 | 应对 |
|---|---|
| GBDT 过拟合 | 严格 train/val/test 分离；用早停、正则化、purged CV |

| 数据质量 | 接入多个数据源交叉验证；异常值检测 |
| 高换手 | 加入换手率惩罚、交易成本；限制再平衡频率 |
| 市场环境变化 | 滚动训练 + regime 感知 + 模型漂移监控 |

## 12. Phase 4 实验总结与下一步（胜率导向）

### 12.1 当前最佳组合一览

下表汇总截至目前所有实验中表现最好的几组配置（h=10，20 只等权，composite-only GBDT）。

| 排名 | 配置 | 累计超额 | 胜率 | 年化夏普 | 平均超额 | 换手率 | 说明 |
|---|---|---:|---:|---:|---:|---:|---|---|
| 1 | **召回过滤 + 宏观二值 -0.3 + WF 阈值** | **+68.20%** | **56.2%** | **0.82** | **+0.41%** | 64.1% | 当前最佳，胜率终于突破 |
| 2 | 召回过滤 + WF 阈值 | +58.60% | 57.1% | 0.75 | +0.37% | 66.0% | 召回层本身即有效 |
| 3 | **等权 + WF 阈值（原 baseline）** | +53.22% | 55.2% | 0.61 | +0.36% | 66.6% | 原生产默认 |
| 4 | 行业权重上限 0.5 + WF 阈值 | +56.86% | 52.4% | 0.65 | +0.39% | 62.9% | 夏普较优，胜率下降 |
| 5 | 宏观二值过滤 threshold=-0.3 + WF 阈值 | +60.03% | 54.3% | 0.66 | +0.39% | 64.2% | 累计超额较高 |
| 6 | 行业权重 0.5 + 宏观二值 -0.3 + WF 阈值 | +62.48% | 51.4% | 0.69 | +0.41% | 60.7% | 夏普较高，胜率最低 |

> 结论：目前所有“增强”都是**用胜率换夏普/累计收益**。胜率天花板仍在 55% 左右。

### 12.2 为什么胜率难突破？

1. **信号源单一**：模型主要依赖量价/财务/资金复合因子，信息高度相关；缺少独立的文本/事件/另类 alpha。
2. **风格切换未被模型捕捉**：宏观/风格特征直接进截面模型会过拟合，组合层择时只能过滤尾部风险，不能提升方向胜率。
3. **预测是回归目标**：模型优化的是收益率大小，不是“跑赢/跑输”的概率；高平均超额不一定等于高胜率。
4. **交易成本与再平衡频率**：10 日再平衡 + 高换手（66%）让小幅优势被成本侵蚀，部分期间由盈转亏。
5. **缺乏“不确定性”过滤**：模型对每笔预测没有置信度概念，所有 top-20 都被同等对待。

### 12.3 下一步重点方向（按预期胜率提升排序）

#### A. 模型集成 + 共识过滤（最可能提升胜率）

- 训练多个 GBDT（不同随机种子、不同特征子采样、或 LightGBM/XGBoost 混合），对 top-20 取**多模型共识**；
- 只有被多数模型选入 top-20 的股票才买入；
- 这能过滤“模型分歧大”的噪声股，降低错误率，直接提升胜率。

**可快速实验**：在 `model_trainer.py` 中加入 `walk_forward_predict_ensemble`。

#### B. 预测置信度 / 信号强度空仓

- 每期计算 top-20 与 bottom-20 的预测得分差（spread）；
- 若 spread 低于过去 N 期滚动中位数，则空仓或减仓；
- 本质是让模型“没把握”时休息，避开震荡期。

#### C. 披露日/事件风险过滤

- 用已有的 `disclosure_date` 数据，剔除“未来 5 个交易日内有业绩预告/财报披露”的股票；
- A 股在财报窗口前后波动大、方向难测，避开这些日期可减少亏损期。

#### D. 组合层面止损 / 回撤控制

- 当组合净值从高点回撤超过阈值（如 5%）时，下一期强制减仓/空仓；
- 个股硬止损已被证伪，但组合层面止损可能截断连续亏损链条。

#### E. 目标函数改为方向分类 + Focal Loss

- 把 GBDT 目标从“超额收益回归”改为“是否跑赢基准的二分类”，并用 Focal Loss 关注难分样本；
- 直接优化胜率，可能把胜率推上 60%，但需验证夏普是否同步改善。

### 12.4 建议优先级

1. **先做 A（模型集成）**：改动相对小，不依赖新数据，且对胜率有直接帮助；
2. **同步做 C（披露日过滤）**：已有 `disclosure_date` 数据，一天可验证；
3. **如果 A+C 仍无法突破 60%，再做 E（方向分类目标）**。

### 12.5 两阶段框架实验（已完成）

新增 `scripts/two_stage_gbdt.py`：
- **召回层**：二分类模型预测“是否值得进入候选池”，输出 `recall_prob`；
- **排序层**：在召回池内再用二分类模型预测“跑赢基准的概率”，输出 `rank_prob`（0~1）；
- 最终持仓按 `rank_prob` 取 top-20。

实现要点：
- `model_trainer.py` 新增 `binary` 目标，支持 LightGBM/XGBoost 二分类；
- 召回池按当日 recall_prob 前 `recall_ratio` 比例截取；
- 排序模型只在召回池内的样本上训练。

**回测结果（h=10，20 只等权，composite-only GBDT）**：

| 配置 | 召回池占比 | 平均超额 | 累计超额 | 胜率 | 说明 |
|---|---|---|---|---|---|
| 单一 GBDT 回归 baseline | 100% | +0.36% | **+53.22%** | **55.2%** | 当前最佳 |
| 两阶段 binary（recall 30% → rank top-20） | ~30% | +0.28% | +39.89% | 48.6% | 召回池太小 |
| 两阶段 binary（recall 50% → rank top-20） | ~50% | — | — | ~47% | 效果更差 |
| 两阶段 binary（recall 30%, rank 目标 top20%） | ~30% | — | — | ~47% | 同样不佳 |

> 结论：
> - 二分类排序模型**没有提升胜率**，反而明显低于回归 baseline；
> - 原因是当前特征集更适合预测“收益大小”（正偏分布）而不是“方向概率”；
> - 另外当前 composite 数据集每日只剩约 80 只股票，召回 30% 只有 23 只，样本太窄，两阶段优势发挥不出来；
> - 如果后续要做两阶段，需要：
>   1. 召回层用**规则**保证覆盖率和质量，而不是再用一个分类模型缩窄；
>   2. 排序层用**专门学习方向概率**的模型，但输入特征需要更多方向性信号（如资金流突变、事件、文本情感）。

### 12.6 召回层规则优化（突破点）

新增 `scripts/recall_filter_trainer.py` 与 `model_trainer._apply_recall_filters`：
- 在训练/预测前按规则过滤股票，保证 GBDT 排序模型只在更干净的池子里学习；
- 规则通过命令行配置，格式为 `col:min:max`。

测试了多组规则，关键发现：

| 召回规则 | 累计超额 | 胜率 | 夏普 | 说明 |
|---|---|---|---|---|
| 无（baseline） | +53.22% | 55.2% | 0.61 | 原默认 |
| `roe>0.2, profit_growth>0.2, vol<0.8, mv>0.2` | -1.05% | 50.5% | 0.01 | 过滤太严，去掉 alpha |
| `vol<0.95` | +20.77% | 49.5% | 0.29 | 单独无效 |
| `mv>0.05` | -34.40% | 43.8% | -0.46 | 单独无效 |
| **`vol<0.95, mv>0.05`** | **+58.60%** | **57.1%** | **0.75** | **最佳召回规则** |
| `vol<0.95, mv>0.05` + 行业权重 0.5 | +47.30% | 52.4% | 0.62 | 行业约束反而拖后腿 |
| **`vol<0.95, mv>0.05` + 宏观二值 -0.3** | **+68.20%** | **56.2%** | **0.82** | **当前全局最优** |

> 结论：
> - 召回层确实能提升胜率。只要**剔除最高波动率 + 最小市值**的尾部（各约 5%），候选池质量就明显提升；
> - 过滤规则不能过严（如强制 roe>0.2），否则会误杀高弹性 alpha 票；
> - 召回规则与组合层宏观择时叠加后，胜率突破到 **56.2%**，累计超额 +68.20%，夏普 0.82；
> - 行业权重约束在这个更干净的候选池上不再有效，说明之前行业集中的问题很大程度上是极端小票/高波动票造成的。
>
> **推荐生产配置**：
> ```bash
> # 训练（召回过滤）
> python scripts/recall_filter_trainer.py \
>   --dataset memory/dataset/features_h10_composite.parquet \
>   --filters "volatility_20:0:0.95,total_mv:0.05:1"
>
> # 回测（召回 + 宏观二值 -0.3 + WF 阈值）
> python scripts/walkforward_gbdt.py \
>   --pred-path memory/predictions/predictions_h10_recall_volatility_20_0_0.95-total_mv_0.05_1.parquet \

>   --macro-dataset memory/dataset/features_h10_composite_phase3.parquet \

> ```

### 12.8 召回层深入规划（胜率导向）

当前 `volatility_20 + total_mv` 规则只是第一步，接下来围绕“提升胜率”可以从以下方向深挖：

#### A. 动态召回阈值（ walk-forward 校准）

- 现在 `0.95 / 0.05` 是固定分位，可能在某些市场环境下过松/过严；
- 改为：每期用过去 N 期的数据，按验证集胜率自动选择最佳 `volatility` / `total_mv` 阈值；
- 避免固定规则在未来失效。

#### B. 多维度质量过滤

- 除了波动率和市值，加入：
  - 流动性（近 20 日成交额 percentile）；
  - 盈利稳定性（ROE 连续为正季度数）；
  - 财务风险（高负债、现金流为负）；
  - 价格异常（近期涨跌停、跳空缺口）。
- 用**轻量规则 + 单因子 IC 验证**，只保留能稳定提升胜率的过滤条件。

#### C. 事件/披露日召回过滤

- 用 `disclosure_date` 剔除未来 5 个交易日有财报/业绩预告的股票；
- 用 AKShare 公告数据剔除近期有重大负面事件的股票；
- 这是**胜率提升最直接**的方向之一，因为事件窗口方向最难预测。

#### D. 风格/行业中性召回

- 在召回层就限制单行业占比、或要求市值/估值分布与基准接近；
- 避免排序模型在某一风格上过度暴露，减少风格切换期的连败。

#### E. 多层次召回（规则 + 分类 + 主题）

- 第一层：硬规则（流动性、市值、ST、价格异常）；
- 第二层：轻量分类模型判断“是否值得进一步评估”；
- 第三层：主题/叙事过滤（如只保留当前政策/资金偏好的行业，需文本数据）。
- 最终召回池 300~500 只，再交给 GBDT 排序。

#### F. 引入“负样本学习”

- 单独训练一个“垃圾股识别模型”：预测某只股票是否属于当日后 20%；
- 召回时直接剔除高概率负样本；
- 这相当于用模型做风险过滤，可能比规则更精准。

### 12.9 召回层实验：财报/预约披露日过滤（已完成）

在 `recall_filter_trainer.py` 中接入 Tushare `disclosure_date` 预约披露时间表，剔除未来 N 个交易日内有财报/季报预约披露的股票。

测试结果（在 `vol<0.95, mv>0.05` 召回 + 宏观二值 -0.3 基础上叠加）：

| 披露窗口过滤 | 剔除比例 | 累计超额 | 胜率 | 说明 |
|---|---|---|---|---|
| 不过滤 | — | **+68.20%** | **56.2%** | 当前最佳 |
| 未来 5 个交易日 | 63.4% | +0.59% | 48.6% | 过度过滤，样本不足 |
| 未来 2 个交易日 | 63.4% | +0.59% | 48.6% | 同样过度过滤 |
| 未来 1 个交易日 | 63.4% | +0.59% | 48.6% | 同样过度过滤 |

> 结论：
> - A 股财报/季报预约披露时间表覆盖的股票比例很高，只要设置 N>=1 就会剔除约 63% 的样本；
> - 过度过滤导致每日候选池只剩 ~26 只股票，模型可学习样本大幅减少，绩效明显劣化；
> - **预约披露日硬过滤不适合作为召回层规则**。
> - 后续若要做事件过滤，应转向：
>   1. 只过滤**业绩预告/快报**（密度低、冲击大）；
>   2. 用 AKShare 公告做**负面语义过滤**（而非所有披露）。

### 12.10 高级召回层实验（动态阈值 / 负样本 / 业绩预告）

新增 `scripts/recall_advanced.py`，同时实现三种高级召回策略：
- **动态阈值校准**：每个 fold 在验证集上搜索最佳 `volatility_20` / `total_mv` 阈值；
- **负样本识别模型**：训练二分类模型预测“是否落入当日后 20%”，剔除高概率负样本；
- **业绩预告/快报事件过滤**：用 Tushare `forecast` / `express` 公告日期，剔除未来 N 个交易日有预告的股票。

测试结果（均在 h=10、20 只等权、WF 阈值、宏观二值 -0.3 基础上）：

| 召回策略 | 累计超额 | 胜率 | 夏普 | 说明 |
|---|---|---|---|---|
| 静态 `vol<0.95, mv>0.05`（当前最佳） | **+68.20%** | **56.2%** | **0.82** | 基准 |
| 动态阈值校准（metric=win_rate） | +72.47% | 51.4% | 0.80 | 累计更高，但胜率下降 |
| 负样本过滤 10% | +56.10% | 55.2% | 0.65 | 无提升 |
| 负样本过滤 20% | +30.45% | 55.2% | 0.41 | 过滤越多越差 |
| 静态 + 负样本 10% | +6.12% | 45.7% | 0.13 | 叠加反而严重劣化 |
| 业绩预告/快报 3 日 | -9.86% | 46.7% | -0.12 | API 限制 + 样本稀疏，结果不可靠 |

> 结论：
> - **动态阈值校准**能让累计超额更高（+72.47%），但验证集上优化的 win_rate 阈值在测试集上表现不稳定，胜率反而降到 51.4%；
> - **负样本识别模型**没有提升胜率，说明当前特征区分“后 20%”的能力不足以帮助排序；
> - **业绩预告/快报过滤**受 Tushare API 限制（需按 ann_date 逐个日期拉取，耗时且数据稀疏），本次结果不可靠，需要更稳定的数据源或缓存机制；
> - 到目前为止，**胜率最高仍是静态 `vol<0.95, mv>0.05` + 宏观二值 -0.3（56.2%）**。

### 12.12 全市场训练实验（2026-07-06）

**问题**：`build_dataset.py` 训练集仅包含 `PASS1_TOP_K=80` 只候选股（由 screen.py pass1 选出），模型从未见过被排除的 4000+ 只股票，存在**训练集选择偏差**。如果 pass1 本身有漏选，模型无法学到正确信号。

**实验设计**：
- 创建 `scripts/build_full_market_dataset.py`：跳过 pass1/pass2，直接从 daily/daily_basic 构造基础量价/估值特征，覆盖全市场 ~4300 只股票（过滤 ST、换手率、流动性）。
- 创建 `scripts/build_full_pass1_dataset.py`：执行 pass1（设置 `AH_PASS1_TOP_K=9999`），保留所有 pass1 存活者（~2500-4500 只/期），使用 pass1 特征但跳过 pass2 财务抓取。
- 对比实验：候选模型（composite 特征，~80 只/期） vs 全市场模型（base 特征，~4300 只/期） vs 全 pass1 模型（pass1 特征，~3000 只/期）。

**实验结果**：

| 数据集 | 行数 | 特征 | Mean IC | 正 IC 比例 | 累计超额 | 胜率 | 说明 |
|---|---|---|---|---|---|---|---|
| 候选模型（composite） | 9,181 | 29 | 0.032 | 58.5% | **+43.93%** | **56.2%** | 当前最佳基线 |
| 全市场 base（recall） | 226,133 | 18 | 0.035 | 62.7% | +8.31% | 48.3% | IC 不错但 top-20 组合弱 |
| 全 pass1 base | 253,262 | 18 | -0.008 | 50.9% | -18.35% | 51.7% | 模型反转 pass1 信号 |

**分析**：
- 全市场 base 模型 IC 0.035 + 63% 正 IC，说明截面排序能力存在；但 top-20 组合仅 +8.31%，因为排名靠前的股票多为**微盘/高波动小票**（缺乏 composite 质量信号），交易成本和滑点吃掉收益。
- 全 pass1 base 模型（加入 pass1_score 特征）训练后预测方向反转，模型学到"pass1 得分越高→未来超额越低"。原因：pass1_score 是 base 特征的线性组合，截面中性化后残差与未来收益的关系可能为负；同时缺乏 composite 特征导致无法区分 pass1 内部质量差异。
- **结论：仅靠 base 特征（动量/估值/市值/波动率）不足以在全市场分布上产生 alpha。composite 质量信号（roe/profit_growth/defensive_quality 等）是当前模型的核心 alpha 来源，必须从 pass2 获取。**

**下一步**：
1. 完成全市场 pass2 数据预取（`prefetch_data.py` 已运行，预计 55 分钟）；
2. 用 `build_dataset.py --universe-size 9999` 构建全 pass1+pass2 数据集（包含 composite 特征）；
3. 在全 pass2 数据集上训练 GBDT，验证是否能提升性能；
4. 如果全 pass2 数据集效果不佳，说明选择偏差不是当前瓶颈，瓶颈在于**特征空间**（需要更多独立信号源）。

### 12.13 集成模型实验（2026-07-07）

**问题**：单一 GBDT 模型 win rate 56.2% 已接近上限，需要多模型集成突破瓶颈。

**实验设计**：
- 创建 `scripts/ensemble_trainer.py`，集成 LightGBM + XGBoost + Ridge 三个模型。
- 两种组合方式：简单平均（average）和 Stacking（用 Ridge 组合）。
- 候选数据集：`features_h10_composite.parquet`，30 个特征。

**实验结果**：

| 方法 | 累计超额 | 胜率 | Mean IC | ICIR | 说明 |
|---|---|---|---|---|---|
| 单一 GBDT | +43.93% | 56.2% | 0.032 | 0.16 | 当前基线 |
| 集成平均 | -13.18% | 40.0% | 0.045 | 0.22 | IC 提升但组合收益下降 |
| **Stacking** | **+44.13%** | **59.0%** | **0.050** | **0.25** | **胜率+2.8%，ICIR+56%** |

**分析**：
- Stacking 用 Ridge 作为 meta-learner，自动学习三个模型的最优权重。
- Mean IC 从 0.032 提升到 0.050（+56%），ICIR 从 0.16 提升到 0.25（+56%）。
- 胜率从 56.2% 提升到 59.0%（+2.8 个百分点）。
- 简单平均效果差，说明模型之间存在负相关或权重不合理。

**结论**：Stacking 集成是有效的提升路径，单一 GBDT 确实是瓶颈。

**下一步**：
1. 加入 CatBoost（修复 deepcopy 问题）；
2. 尝试动态集成（滚动窗口选择近期表现最好的模型）；
3. 加入 MLP 神经网络作为第四个基模型；
4. 尝试双层 Stacking（Level 0: GBDT 家族, Level 1: 线性模型）。

**CatBoost 修复 & 4 模型 Stacking（2026-07-07 更新）**：

修复方案：使用 `CatBoostRegressor` 类替代 `cb.train()`，避免 Pool 对象 deepcopy 问题。

CatBoost 单独 Mean IC 仅 0.016（最弱），但 Stacking 自动降低其权重，整体胜率从 56.2% 提升到 **60.0%**（+3.8%）。
